# Changelog

本文件记录 GitHub 仓库的版本更新。每次发布前，先补充对应版本的新增、修改和修复内容。

## [2.1.4] - 2026-07-22

### 新增

- 标准文字套版支持为副标题中的指定文字片段设置独立颜色：在 `material.json` 的 `style.subtitleHighlights` 中配置文字与颜色，renderer 会按片段直接绘制高亮字色。

## [2.1.3] - 2026-07-22

### 新增

- 原图迁移流程新增“台球杆纯商品图”QA 例外：完整执行并记录 QA；候选图可用且非 `BLOCKED` 时，将偏差记录为观察项后直接渲染，不再触发生图重试。
- 标准文字套版新增台球杆右偏构图与参考图边缘出画约束，支持在保留右上出画关系的前提下进行裁切、测量与重试。

### 修改

- 标题与副标题识别仅在文字不清晰、跨多行或层级存在歧义时要求用户确认；确认前不创建任务数据或生图。
- 台球杆及其他边缘出画主体的 QA 改为测量可见轮廓并核验出画关系；重试提示改用参考图可观察到的实体接触、阴影和遮挡关系作为保真约束。

## [2.1.2] - 2026-07-21

### 新增

- `channel` 与 `categoryBanner` 的首轮横图若将核心主体截在白色缓冲区边界，改走一次连续场景背景恢复：记录完整主体边界，并按主体垂直中心生成最终裁切。
- `prepare-channel-input.py` 新增 `--main-visual-bounds` 参数，支持使用视觉 QA 测得的主体边界生成横向资源输入。

### 修改

- 所有包含 `feed`、`popup` 或 `splash` 的请求统一生成并复用一张 `9:16` 竖版开屏母图，再按各资源位派生裁切。
- 竖版重试提示增加保持主体组原有构图关系的约束；横向恢复候选仍有裁切或平衡问题时，以 `RENDER_WITH_OBSERVATION` 记录后继续渲染。

## [2.1.1] - 2026-07-21

### 新增

- 支持无主、副标题的 `1035:390`（`2.65:1`，允许 1% 比例误差）横图直接路由至 `push`，跳过 AI 生图与额外确认，直接交由 renderer 套版。

## [2.1.0] - 2026-07-20

### 新增

- 无主、副标题的普通商品图、商品组图和场景图新增“标准文字套版”路由：先生成一张 `9:16` 开屏母版，再派生 `feed`、`popup` 和 `splash` 成图。
- 支持同一张无标题 `9:16` 开屏底图作为垂直母版，一次套版生成一个或多个 `feed`、`popup`、`splash` 资源位，并记录主体垂直中心用于裁切。

### 修改

- 仅已完成、具备连续文案安全区的无标题 `9:16` 开屏底图可跳过生图直接套版；透明抠图、白底/纯色底商品图及普通场景图不再因比例匹配而直接渲染。
- `feed` 和 `popup` 的无标题标准文字套版统一复用 `9:16` 开屏母版，保证不同竖版资源位的主体位置与文案安全区一致。
- 尺寸匹配脚本的直接路由收敛为横向资源与开屏母版，避免将 `3:4` 或 `1:1` 的普通源图误判为可直接套版底图。

## [2.0.0] - 2026-07-20

### 重大变更

- 根目录 `SKILL.md` 改为物料流程分发器：先根据当前图片是否含有可辨识的主、副标题，路由到 `原图迁移` 或 `文字模板` 工作流。
- 新增 `source-style-migration/`：保留参考图主、副标题的字体与视觉样式，并覆盖频道、分类 Banner、Push、feed、弹窗、开屏及小程序分享等资源规范。
- 新增 `standard-renderer-copy/`：清除图内主、副标题后，使用标准 renderer 模板绘制标题、副标题与 CTA。
- 字体统一迁移到根目录 `assets/fonts/`，两条子工作流共用，避免重复副本。

### 流程调整

- 含主、副标题的图片必须由用户选择“原图迁移”或“文字模板”；无主、副标题且已提供标题、副标题时，直接进入文字模板流程。
- 分发器只负责检查与路由，生产提示、QA、renderer 和资源规则全部由被选中的子工作流执行。
- 原图迁移工作流补充了资源位布局指南与小程序分享资源；标准文字模板工作流保留现有的比例路由、QA 门禁和 renderer 流程。

### 兼容性说明

- 这是不兼容的结构性升级；`v1.5.0`（`d71aec0`）仍完整保留在原发布分支，可随时切回。

## [1.5.0] - 2026-07-19

### 新增

- 为 `feed`、`popup` 的定向重生新增测量预检：必须记录最终裁切映射、完整主体像素边界、缩放计算及相对主视觉区的位移后，才能生成重试提示。
- 明确首次生图提示的最小结构：参考母图承载场景与风格，提示仅补充资源位构图、安全区、原主文案清除和模板安全要求。
- 定向重试新增对主体与承托物的接触、遮挡、接触阴影及受力逻辑的保留要求。

### 修改

- `feed`、`popup` 区分目标构图边界与 QA 下缘容差：非关键下缘可在限定范围内被 QA 接受，但不会扩大主视觉或 CTA 的布局区域。
- renderer 的主体适配计算改用 QA 接受边界；CTA 本体仍不得包含任何主体、关键结构或品牌识别内容。
- 更新初始生成、视觉 QA、最终验收和渲染流程文档，使其统一使用“容差仅用于 QA 判定”的规则。

### 修复

- 修复首次生成提示重复描述参考母图场景与风格，增加模型偏离参考图风险的问题。
- 修复下缘容差被误解为 CTA 可放置主体内容的布局区域的问题。

## [1.4.0] - 2026-07-19

### 新增

- `feed`、`popup` 新增 schema v3 的像素级主体适配门禁：基于最终渲染画面测量主体边界、计算所需缩放比例，并校验 `mainVisualFits` 结论。
- 支持无主标题/副标题图片按资源位比例直接路由：`3:4` 至 `feed`、`1:1` 至 `popup`、`9:16` 至 `splash`、约 `4.63:1` 至 `channel`，无需先匹配最终输出像素尺寸。
- 为 `feed`、`popup` 补充按测量值生成缩放与居中定向重试文案的完整协议。

### 修改

- 新任务统一使用 `qaSchemaVersion: 3`；`feed`、`popup` 除居中测量外还需提供 `fitMeasurement` 与 `mainVisualFits`。
- 现有无主文案背景可依据支持比例直接交给 renderer 居中 cover 裁切，不再要求预先本地裁切到最终尺寸。
- 更新 `feed`、`popup`、`splash` 的主体组表述和 QA 说明，使其按最终 renderer 裁切进行判断。

### 修复

- 修复非最终像素尺寸但比例正确的无主文案图片被错误要求重新生图的问题。
- 修复主体尺寸与 CTA/文案安全区冲突时缺少可验证缩放计算的问题。

## [1.3.0] - 2026-07-19

### 新增

- 当参考图没有有效环境背景（如透明抠图、纯色底或黑白底孤立商品）时，默认使用从商品主色提取的 `极简光影底`，为文案提供低饱和环境光、渐变或柔和地面反射。
- 为 `feed`、`popup` 明确顶部全宽文案安全区与底部 CTA 硬性避让区的比例和验收要求。
- 新增以失败候选图为输入的单句定向重试格式：仅描述实际核心主体、测得的修正动作、目标与保留约束。

### 修改

- `feed`、`popup` 的 CTA 区调整为硬性避让：不得出现任何主体内容、商品结构、可读文字、Logo 或品牌核心识别信息。
- 定向重试优先按主视觉居中测量修正；仅在主体已经垂直居中且尺寸造成 CTA 占用时才允许轻微缩小。
- 更新文案承托、背景生成、预渲染 QA、渲染流程与最终检查清单，使其一致采用自然过渡的安全区和硬性 CTA 验收。

### 修复

- 修复无环境背景参考图仍按“默认原图”处理时，缺少与商品一致的文案承托和环境表达的问题。
- 修复 CTA 占用时仅以避让 CTA 为目标而偏离主视觉居中规则的问题。

## [1.2.0] - 2026-07-19

### 新增

- 为 `feed`、`popup`、`splash` 新任务新增 `qaSchemaVersion: 2` 和 `centeringMeasurement`：以主体边界和中心点量化校验主视觉居中，避免仅凭主观判断标记通过。
- 明确 AI 生成背景画布比例：`feed` 为 `3:4`、`popup` 为 `1:1`、`splash` 为 `9:16`；由 renderer 执行居中 cover 裁切。
- 补充 `feed`、`popup` CTA 可读性与非关键内容轻微重叠的容忍标准，以及按实际核心主体名称生成的定向修正语句。

### 修改

- 移除 5 张版式参考图依赖，改为将资源位的画布、留白、主体位置和安全区规则直接写入生成提示与 QA 规范；参考母图成为唯一图片输入。
- 渲染器在 schema v2 下校验居中测量数据的格式、数学一致性和 `mainVisualCentered` 结论的一致性。
- 统一 `RENDER_WITH_OBSERVATION` 后的资源输入准备流程，并允许分类 Banner 在该状态下完成裁切与渲染。
- 更新背景生成、文案安全、渲染、视觉 QA 与最终检查清单，使其与固定比例和可量化居中规则一致。

### 修复

- 修复依赖版式参考图时，生成模型可能将参考图结构当作视觉素材或忽略实际画布比例的问题。
- 修复 CTA 轻微覆盖非关键区域时被一律判定失败，以及主体居中结论缺乏可验证依据的问题。

## [1.1.0] - 2026-07-18

### 新增

- 渲染器新增 `material.json.preRenderQA` 强制门禁：每个请求资源均需提供完整的预渲染 QA 记录、检查项和证据后才可输出 PNG。
- 新增 `RENDER_WITH_OBSERVATION` 状态：定向重试一次后仍存在问题时，可携带具体观察项继续渲染，并在最终交付中明确说明。
- `feed`、`popup`、`splash` 新增统一的构图提示写法和主视觉水平、垂直居中规范。
- 新增基于实际 QA 偏移的定向重生规则，以及 `feed`、`popup` 的 CTA 避让决策顺序。

### 修改

- `channel` 的预渲染 QA 改为建议性流程；即使存在 `ADVISORY_FAIL`，也会保留观察项并继续执行预处理和渲染。
- 频道预处理脚本改为根据检测到的核心主体组垂直中心选择裁切位置，提高最终横图主体居中表现。
- 统一六类资源的 QA 检查项、渲染前校验、渲染流程和最终交付检查清单。
- `splash` 不再以底部操作区作为 QA 或定向重生条件，聚焦标题安全区与主视觉居中。

### 修复

- 修复预渲染 QA 失败后无法保留具体问题并完成可用物料输出的流程断点。
- 修复 `feed`、`popup`、`splash` 主视觉默认偏上而非居中的规范偏差。

## [1.0.0] - 2026-07-18

### 主要功能

- 支持 `channel`、`categoryBanner`、`push`、`feed`、`popup`、`splash` 六类标准运营资源位。
- 支持从参考图生成模板安全底图，并通过资源位规则、文案安全检查和快速视觉 QA 后渲染。
- 提供 Python/Pillow renderer，内置阿里妈妈数黑体主标题、阿里巴巴普惠体副标题与按钮，并校验成图技术规格。
- 默认将工作输入与输出重定向到 skill 外的工作区，避免将任务文件带入 skill。

### 仓库范围

- 仅提交运行 skill 所需的规则、脚本、字体与 Agent 元数据。
- 不提交本地任务输入/输出、Finder 元数据、同事说明或本地导出辅助脚本。
